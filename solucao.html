A Solução Zapdos: Uma Estratégia de Três Pilares para a Integridade Total do Ativo

O Projeto Zapdos é concebido como um sistema holístico e integrado que aborda o desafio "Motor sem traumas no transporte" de forma abrangente. A solução não se limita a um único aspecto do problema, mas implementa uma estratégia de três pilares interconectados que cobrem a prevenção, a detecção e a validação, garantindo a integridade do ativo desde a fábrica até à instalação final.


Pilar I: Isolamento Mecânico Proativo (Prevenção)

Este pilar constitui a primeira linha de defesa e aborda diretamente o primeiro ponto do desafio: "Travamento/isolamento do eixo e base de amortecimento para transporte". O objetivo é redesenhar fundamentalmente a interface física entre o motor e o veículo de transporte para mitigar a causa raiz do problema.


Base de Amortecimento Adaptativa


O segundo componente deste pilar é a engenharia de uma nova base de transporte que funciona como um sistema de amortecimento ativo. Em vez de uma simples palete ou estrado, esta base será equipada com suportes de isolamento de vibração, como coxins elastoméricos ou outros materiais com propriedades de amortecimento específicas. Estes materiais serão selecionados para absorver energia nas faixas de frequência problemáticas, tipicamente encontradas no transporte rodoviário. O objetivo é isolar o motor das vibrações do veículo, impedindo que essa energia destrutiva chegue aos rolamentos.

Pilar II: Monitorização da Jornada em Tempo Real (Detecção)

O segundo componente deste pilar é a engenharia de uma nova base de transporte que funciona como um sistema de amortecimento ativo. O Núcleo do Sensor IoT


O coração deste pilar é uma unidade de sensor IoT de baixo custo, robusta e autónoma. O hardware central será uma Unidade de Processamento de Movimento (MPU) ou Unidade de Medição Inercial (IMU), que integra um acelerómetro de 3 eixos e um giroscópio. Este sensor será montado diretamente na carcaça do motor, próximo ao alojamento do rolamento, para capturar os dados de vibração mais relevantes. A unidade também incluirá um módulo GPS para correlação de eventos com a localização (por exemplo, "evento de alto choque ocorreu nas coordenadas X, Y, numa estrada em mau estado") e um modem celular para a transmissão de dados, criando uma "Rede IoT" eficaz.

Arquitetura de Dados e Plataforma na Nuvem


Para gerir o fluxo de informações, será implementada uma arquitetura de dados escalável. O dispositivo IoT registará os dados localmente com carimbos de data e hora e realizará uploads periódicos ou acionados por eventos para um servidor na nuvem. Os dados serão armazenados numa base de dados de séries temporais, otimizada para consultas e análises de grandes volumes de dados de sensores. Esta abordagem demonstra uma visão de futuro sobre a gestão e o aproveitamento dos dados recolhidos.

Pilar III: Análise Preditiva de Dados (Validação)

Este é o pilar mais inovador e aborda o terceiro e mais complexo ponto do desafio: "Análise dos dados coletados e critérios de aceitação (níveis de energia necessários para causar dano)". Este pilar transforma dados brutos em inteligência acionável, utilizando conceitos avançados de processamento de sinais e modelagem preditiva.


Da Dado Bruto à Correlação de Danos


A metodologia começa com o processamento dos dados brutos do acelerómetro. Serão extraídas características estatísticas chave tanto no domínio do tempo (força G de pico, aceleração RMS, curtose, fator de crista) como no domínio da frequência (utilizando a Transformada Rápida de Fourier - FFT - para identificar o conteúdo energético em bandas de frequência específicas). Este processo fornece um "método para obter força do impacto confiável", conforme idealizado pela equipa.

Modelo Quantitativo de Limiar de Dano


O núcleo analítico do projeto é a correlação dos dados da jornada (Pilar II) com os dados de inspeção pós-viagem (como a análise BPFI do estudo de caso). Ao analisar múltiplos transportes, será construído um modelo preditivo que liga perfis de vibração específicos à probabilidade de ocorrência de falso brinelamento. O objetivo final é definir um limiar de dano claro e baseado em dados. Este limiar pode ser expresso como um "Valor de Dose de Vibração" ou uma métrica de energia de choque cumulativa (medida em Joules, conforme sugerido pela menção a "níveis de energia" no desafio). O resultado será um conjunto de critérios de aceitação simples e acionáveis. Por exemplo: "Se a energia de vibração cumulativa exceder X Joules, o motor é automaticamente sinalizado para uma inspeção obrigatória do rolamento antes da entrega ao cliente, independentemente da sua aparência funcional." Este sistema cria um ciclo de feedback virtuoso: os dados do Pilar II, analisados pelo Pilar III, fornecem uma avaliação quantitativa e direta do desempenho do Pilar I. Se uma determinada rota gera consistentemente níveis de vibração próximos do limiar de dano, esses dados podem ser usados para reforçar a solução mecânica ou para aconselhar o parceiro logístico a utilizar uma rota alternativa. Isto transforma o sistema de um simples monitor num instrumento de gestão logística inteligente e de melhoria contínua.
